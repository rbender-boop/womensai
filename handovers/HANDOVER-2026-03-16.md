## What Was Built This Session

### 1. Habit-First Signup Prompt System
A three-phase signup prompt that lets users build reliance before asking for anything.

**Files:**
- `components/signup-prompt.tsx` — UI component with 3 variants: `banner`, `modal`, `return`
- `hooks/use-signup-trigger.ts` — All trigger logic using localStorage

**Trigger logic:**
- Question 3+ → soft dismissable **banner** slides up from bottom
- Question 5 (limit hit) → **modal**
- 3rd return session + 3+ questions → **"Welcome back" modal**
- Dismissed → won't show again for 4 hours
- Once signed up → all prompts stop permanently

**localStorage keys used:**
- `wai_q_count` — total questions asked lifetime
- `wai_sessions` — number of distinct sessions
- `wai_last_session` — timestamp of last session start
- `wai_dismissed` — timestamp of last dismissal
- `wai_signed_up` — '1' if user has signed up

### 2. Signup API + Supabase
**File:** `app/api/signup/route.ts`

- POST `/api/signup` with `{ email: string }`
- Upserts to Supabase `email_signups` table
- Checks for existing email — only sends welcome email to new signups
- Returns `{ success: true }` even on DB error (never blocks UX)

**Supabase migration:** `supabase/migrations/20260316_email_signups.sql`
- Must be run manually in Supabase SQL Editor if not yet applied
- Creates `email_signups` table with `id`, `email` (unique), `signed_up_at`, `source`

### 3. Resend Welcome Email
**File:** `lib/email.ts`

- Sends from `Kelly at AskWomensAI <kelly@askwomensai.com>`
- Subject: `"You're in — your answers just got personal"`
- Styled HTML email matching site brand (rose/cream palette)
- CTA button links back to askwomensai.com
- Requires `RESEND_API_KEY` env var in Vercel

**Resend setup:**
- Both `womensai.com` and `askwomensai.com` domains are verified in Resend
- No inbox setup needed — Resend sends outbound from any address on verified domains
- To receive replies, would need Google Workspace or Zoho Mail (not set up yet)

## Key Product Decisions This Session

**Signup philosophy:** Habit-first. Don't ask until leaving feels like losing something. Let the product earn the ask over 5-10 questions and 2-3 return visits.

**Hook line (approved):** "Sign up free — get answers personalized to your health history, not just your last question." — then revised to avoid "health history" language → now "the more you ask, the smarter your answers get."

**No pricing in nav** — don't allude to cost until users are hooked.

**Tone:** The AI speaks in first person ("I get smarter...") to feel personal and direct.
