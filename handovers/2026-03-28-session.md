## Known Issue: Unicode in GitHub API

**Root cause:** When pushing file content through the GitHub MCP `push_files` or `create_or_update_file` tools, `\u` escape sequences in raw template literal strings get stored literally instead of being interpreted as Unicode. 

**Workaround applied:** Wrap all user-visible strings containing Unicode escapes in JSX expression syntax: `{'text with \u2014 dash'}` instead of bare `text with \u2014 dash`. Inside a JSX expression (`{}`), JavaScript interprets the escape at runtime. In bare JSX text, the GitHub API stores it literally.

**For future sessions:** Always use this pattern for special characters, or use actual Unicode characters when editing files locally via Desktop Commander.
